Strategic Implication: 3 Platform AI Capabilities (from H8)

H8 applied the H1 cluster taxonomy as cross-domain weights to 24 AI opportunities across 8 products. The result: the AI roadmap should be organized by cluster, not by product.

1. Data Synthesis & Reconciliation
8/9 products · Clusters 9+10 (Data Fragmentation + Manual Correction Burden)
Cross-source entity resolution, fuzzy matching, variance detection. Every product is independently building this. Should be platform infrastructure with per-product adapters.
MVP: PT's Fuzzy-Match Entity Resolver → shared service
2. Trust / Retrieval-First AI Infrastructure
7-8/9 products · Clusters 0+2+7 (AI Ambivalence + Confidence + Navigation Friction)
Source citation, methodology provenance, hallucination prevention. Per-product RAG builds are duplicated effort. Should be a shared RAG framework all AI features run on top of.
MVP: PT's Retrieval-First AI Architecture → platform-wide standard
3. NL Workflow Automation
7/9 products · Clusters 5+6+9 (Urgency + Tool Friction + Data Fragmentation)
Natural language query → synthesized report. Reduces 60-90 min workflows to 5-10 min universally.
MVP: MASS's NL Query Interface → generalized platform capability
The 4 demoted items (Multi-Parameter Candidate Screening, Literature Summarization, Chemistry-Biology AI, Evidence Change Alerting) are product-specific investments that should NOT receive shared platform resources.

Key Algorithm: Pain-Modulated Confidence (H4)

Blind frequency-based confidence weighting breaks high-stakes, low-frequency signals. The correct algorithm:

def confidence_floor(pain_score):
  if pain_score >= 8.0: return 0.9 # protect high-pain signals regardless of frequency
  elif pain_score >= 6.0: return 0.6
  return 0.3

# Example: MENA/Brazil Coverage
# pain=9.0, n_interviews=3 → floor=0.9 → stays P0 ✅
# Blind weighting: 3/10 users → P4 ❌ (false demotion)

Technical Lessons

1
UMAP dramatically outperforms PCA for qualitative clustering — 49% noise → 12% noise on 661 FEELS entries. Always use UMAP+HDBSCAN for semantic clustering of empathy data, not PCA+KMeans.
2
Transcript length does not predict empathy map quality — CDDI has the longest transcripts (7,982 words avg) but the fewest FEELS entries (5.3 avg). Quality is 100% prompt-driven.
3
The (?=\n##|\Z) regex collapse bug — Non-greedy (.*?) with re.DOTALL collapses to 0 chars when FEELS is followed by ### subsection headers. Position-based extraction is the fix.
4
Node ID staleness in Figma is architectural, not a bug — Plugin API doesn't persist node references across separate figma_execute calls. The only fix is single execution; retry loops don't work.
5
Prediction direction is usually right, magnitude is usually off — H1 predicted 5-8 clusters, found 14. H4 predicted 25-35% reclassification, found 43%. The platform is consistently MORE interesting than predicted.
6
Auto-discovery mode is the correct skill design pattern — Skills with auto-discovery (H7 pain-matrix, H9 empathy-clustering) are adopted; skills requiring manual file discovery are not.
